# YOLO26s-Seg 书脊实例分割训练报告

> 本文档记录图书书脊实例分割项目的完整训练过程、参数配置、实验结果，供论文撰写参考。
> 所有数据均来自训练日志和评估结果，未做任何虚构。

## 1. 数据集

### 1.1 数据采集

| 项目 | 数值 |
|------|------|
| 书架照片总数 | 517 张 |
| 原始分辨率 | 4080 × 3072 |
| 标注多边形总数 | 12,678 个 |
| 平均每图多边形数 | 24.5 个 |
| 标注格式 | Labelme 多边形 JSON |

照片拍摄于真实书架场景，包含密集排列的书籍，涵盖不同光照、反射、倾斜和遮挡条件。

### 1.2 标注流程

1. **SAM3 自动标注**：使用 Labelme 内建的 OSAM（Open SAM）自动化管线，对每张书架照片自动生成书脊多边形分割掩码。参数：score_threshold=0.01, iou_threshold=1.0, max_annotations=1000。
2. **Labelme 人工修正**：在 SAM 自动标注基础上，人工检查并修正边界不准、漏标、多标等问题。

### 1.3 标注质量

| 指标 | 旧标注（297图） | 新标注（220图） | 全量（517图） |
|------|----------------|----------------|--------------|
| 平均点数/多边形 | 14.8 | 21.4 | 18.1 |
| ≤5点粗糙标注占比 | 35.1% | 11.0% | 20.3% |
| 多边形总数 | 6,902 | 5,776 | 12,678 |

新标注的多边形点数显著增加（14.8 → 21.4），粗糙标注（≤5点）大幅减少（35.1% → 11.0%），标注质量明显提升。

### 1.4 数据集划分

采用 train/val/test 三分，test 集为独立 holdout（从未参与训练和验证），用于评估真实部署效果。

| 集合 | 原图数 | 总图数（含crop） | 多边形数 | 说明 |
|------|--------|-----------------|---------|------|
| train | 423 | 9,320 | ~10,000 | 80% 原图 + 全部 crop |
| val | 94 | 3,131 | ~2,600 | 20% 原图 + 对应 crop |
| test | 30 | 744 | 1,427 | 独立 holdout，固定不变 |
| **总计** | **517** | **13,195** | **12,678** | |

### 1.5 数据增强策略

训练集采用 **原图 + 掩码 crop 双路**策略：

- **原图**：完整书架照片，模型学习多书脊密集场景
- **掩码 crop**：按标注多边形裁剪单本书脊，背景保留真实书架像素（非纯黑），模型学习单脊精细边界

crop 图的 mask 空间分辨率远高于原图（书脊宽 ~30px vs 原图 ~5px），对 mask mAP50-95 有显著贡献。

### 1.6 在线增强参数

| 增强方式 | 参数 | 作用 |
|---------|------|------|

| HSV 颜色扰动 | h=0.015 s=0.7 v=0.4 | 模拟光照变化 |

| 水平翻转 | 0.5 | 左右翻转 |
| 垂直翻转 | 0.5 | 上下翻转（书脊可倒放） |

## 3. 训练历程

本项目经历两版训练，数据量从 297 张标注扩展到 517 张标注。

| Batch size | 8 |
| 达峰 epoch | 97 |
| 是否早停 | 否（100 epoch 跑满） |
| 显存峰值 | 6.89G |

### 3.2 第二版（v2：517 标注，最终版）

| 项目 | 值 |
|------|-----|
| 标注原图 | 517 张（+220 张新标注） |
| 多边形总数 | 12,678 个 |
| 总训练图 | 13,195 张 |

| 达峰 epoch | 97 |
| 是否早停 | 否（100 epoch 跑满） |
| 显存峰值 | 5.68G |

### 3.3 版本对比

| 指标 | v1 | v2 | 变化 |
|------|-----|-----|------|
| 数据量 | 7,199 图 | 13,195 图 | +83% |
| 多边形数 | 6,902 | 12,678 | +84% |
| Batch size | 8 | 6 | -25%（OOM） |
| val Mask mAP50-95 | 0.994 | 0.991 | -0.003（val 集变大 2.5x） |
| **holdout Mask mAP50-95** | **0.921** | **0.927** | **+0.006** |
| holdout Box mAP50-95 | 0.943 | 0.950 | +0.007 |
| holdout Box Recall | 0.975 | 0.985 | +0.010 |
| holdout Mask Recall | 0.978 | 0.988 | +0.010 |

**关键结论：**

1. **holdout Mask mAP50-95 提升 +0.006**（0.921 → 0.927），在固定 30 张测试集上真实提升
2. **Recall 提升显著**（+0.010），新数据让模型检出更多之前漏检的书脊
3. val 指标下降 0.003 是正常的——val 集从 1,244 图扩大到 3,131 图，测试范围更大
4. 数据量增加 83% 带来的精度提升边际递减，因 0.921 已处于较高水平

### 3.4 收敛分析

两版训练均呈现以下收敛特征：

- **ep 1-10**：快速上升期，Mask mAP50-95 从 ~0.70 升至 ~0.93
- **ep 10-50**：稳步提升期，从 ~0.93 升至 ~0.98
- **ep 50-97**：微调收敛期，从 ~0.98 升至 ~0.99（v1）/ ~0.991（v2）
- **ep 97-100**：平台期，指标在小数点第三位波动
- **达峰 epoch 均为 97**，100 epoch 设置合理

训练损失（v2 最终值）：
- box_loss: 0.2487
- seg_loss: 0.2748
- cls_loss: 0.1100
- dfl_loss: 0.0017

## 5. 馆藏匹配

### 5.1 匹配流程

书脊分割 → OCR 书名识别 → 馆藏目录模糊匹配 → 去重计数

- OCR：Qwen3-VL-32B 多模态模型，识别书架照片中所有书脊的书名和数量
- 馆藏目录：185,976 条书名（清洗后）
- 匹配引擎：rapidfuzz（比标准库快 ~100x）

### 5.2 9 级匹配策略

按优先级从高到低执行，命中即停止：

| 策略 | 分数 | 规则 | 命中数 |
|------|------|------|--------|
| exact_full | 1.0 | OCR 全名 == 馆藏（忽略大小写） | 1,797 |
| exact_main | 0.98 | 主标题 == 馆藏 | 936 |
| prefix_main | 0.92 | 馆藏以主标题开头，后缀≤容忍度 | 92 |
| contained | 0.88 | 馆藏是主标题子串 | 16 |
| contained_in | 0.85 | 主标题是馆藏子串（馆藏有前缀） | 16 |
| partial_full | ≥阈值 | OCR 全名 partial_ratio | 269 |
| token_set | ≥阈值 | token_set_ratio（空格/词序不敏感） | 4 |
| partial_main | ≥阈值 | 主标题 partial_ratio | 63 |
| global_ratio | ≥0.70 | 全库 fuzz.ratio 兜底（不限首字符） | 39 |

### 5.3 主标题提取

从 OCR 书名中去除版本号、副标题、丛书名、编者等噪声，提取主标题用于匹配：

- 去末尾版本号：`（第X版）` `（下卷）` + 裸版本号 `第X版` `上册`
- 去前缀年份：`2008` `2017`
- 去前缀丛书名：`冲击波英语` `（全新题型）` `新编` `插图本`
- 破折号/括号/冒号分隔
- 空格分隔（仅中文书名）

### 5.4 匹配结果

| 指标 | 值 |
|------|-----|
| OCR 书名条目 | 3,342 |
| 已匹配 | 3,232 (96.7%) |
| 需人工确认 | 98 |
| 未匹配 | 110 |
| 总册数 | 7,031 |
| 独立书名 | 2,641 |
| 匹配耗时 | 12.4s（全量）/ <0.5s（单张） |

### 5.5 优化历程

匹配率从 95.4% 提升到 96.7%，主要优化：

1. **统一馆藏目录**：API 和 CLI 统一读取清洗后的 titles.json
2. **prefix_main 误匹配修复**：短主标题分级限制后缀长度，纯英文短词不参与
3. **候选集首字符容错**：首字符无候选时放宽到第 2 字符
4. **token_set_ratio**：处理空格/词序差异
5. **contained_in 双向匹配**：馆藏有前缀的情况（如"新编XXX"）
6. **全库 fuzz.ratio 兜底**：救回 OCR 首字错的书名
7. **主标题提取增强**：去年份/丛书名前缀 + 裸版本号
